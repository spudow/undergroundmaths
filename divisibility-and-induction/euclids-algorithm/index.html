Euclid's algorithm

Here’s an interesting thought. If we’re told that 122381=3×35889+14714122381=3×35889+14714 and that 77 divides both 3588935889 and 1471414714, then we know immediately that 77 also divides 122381122381.

Why is that? Try to find your own explanation.

We can use this idea to understand Euclid’s algorithm. You may have already explored Euclid’s algorithm in the Picture this! investigation.

Let’s run the algorithm on the numbers 126126 and 4949. We start by dividing 126126 by 4949, to find the quotient (in this case 22) and the remainder (in this case 2828). We then repeat the process for the numbers 4949 and 2828, and so on. Here’s the list of equations that we get.
126492821=2×49+28=1×28+21=1×21+7=3×7126=2×49+2849=1×28+2128=1×21+721=3×7

We stop there because the remainder in the final equation is 00.

Here’s a handy tip. Numbers that do the same job are written in the same column. For example, the quotients are all just before the ×× sign, and the remainders are all at the end. There’s a lot of flexibility about the order in which we write the components of these equations, but it’s a lot easier to keep track of which numbers are doing which jobs if we keep them in the same place each time.

What can we deduce from these equations?

The last non-zero remainder is 77, so we’d like to show that the highest common factor of 126126 and 4949 is 77. If you worked on Picture this!, then you might have been able to predict this: the relevant diagram is

To show that 77 is the highest common factor of these two numbers, we need to show two things:

  • that 77 really is a common factor of them (it divides both numbers);

  • and that it’s the highest such number (any common factor of 126126 and 4949 is less than or equal to 77).

We’ll show these things in turn.

77 is a common factor of 126126 and 4949

Of course, we could simply divide each number by 77 and check that we get an integer, but in fact we don’t need to do any more calculations: the equations above contain all the information we need. Let’s see how.

Let’s first start at the bottom and then work up.

The last equation, 21=3×721=3×7, tells us that 77 divides 2121.

The next equation up, 28=1×21+728=1×21+7, then tells us that 77 divides 2828 (because it divides both 2121 and 77 on the right-hand side).

The next equation up, 49=1×28+2149=1×28+21, then tells us that 77 divides 4949 (because it divides both 2828 and 2121 on the right-hand side).

Finally, the top equation, 126=2×49+28126=2×49+28 tells us that 77 divides 126126 (because it divides both 4949 and 2828). In particular, 77 is a common factor of 126126 and 4949.

You might reasonably think that it would be a lot easier just to check directly that 77 divides both numbers, but the advantage of what we’ve just done is that we can generalise it to any pair of starting numbers and still know that the last non-zero remainder divides both starting numbers.

Any common factor of 126126 and 4949 is less than or equal to 77

Now let’s suppose that we have any old common factor of 126126 and 4949. Let’s call it dd.

It’s much easier to talk about something if we’ve given it a name, and dd is a good name for a divisor.

Then the top equation, which we can rewrite as 28=126−2×4928=126−2×49, tells us that dd divides 2828 (because it divides both 126126 and 4949).

The next equation down, which we can rewrite as 21=49−1×2821=49−1×28, tells us that dd divides 2121 (because it divides both 4949 and 2828).

Finally, we can rewrite the penultimate equation as 7=28−1×217=28−1×21. This tells us that dd divides 77 (because it divides both 2828 and 2121).

So if dd is any common factor of 126126 and 4949, then it divides 77, and so is less than or equal to 77.

Conclusion

We have shown that any common factor of 126126 and 4949 is at most 77, and also 77 is a common factor of 126126 and 4949. But that means precisely that 77 is the highest common factor of 126126 and 4949.

We do not have to write out all of that explanation every time we use Euclid’s algorithm. But it’s important that we’re able to explain why the algorithm finds the highest common factor.

Pick your own pair of positive integers, and run Euclid’s algorithm on them. Now explain to a friend why your equations have found the highest common factor for you.

Here are some follow-up questions to help you develop your understanding of Euclid’s algorithm.

  1. Find a pair of integers that have highest common factor 1212 and where it takes exactly four steps of Euclid’s algorithm to find that highest common factor.

    Can you still do this if you change the highest common factor and/or the number of steps?

  2. Find a pair of four-digit numbers for which it takes very few steps for Euclid’s algorithm to find their highest common factor.

  3. Find a pair of four-digit numbers for which it takes many steps for Euclid’s algorithm to find their highest common factor.
