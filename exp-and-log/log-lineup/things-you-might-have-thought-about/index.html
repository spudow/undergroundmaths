Things you might have thought about

Can you, without using a calculator, arrange these logarithms into order of increasing value?

log45log25log34log33log35log24log23log4⁡5log2⁡5log3⁡4log3⁡3log3⁡5log2⁡4log2⁡3

Initial observations

  • We can write down exact values for two of these logarithms.
  • Since log35log3⁡5 is the power of 33 that would give us 55 and 31<5<3231<5<32 we know the value of log35log3⁡5 must lie between 11 and 22. Can you do the same for other logarithms in the list and use the results to put some of them in order?

Increasing or decreasing?

If we take a series of logarithms all with the same base, we might assume that loglog of a bigger number will have a bigger value. In other words, the function logxlog⁡x is increasing.

Is this always true, or only for some values of xx? Does it depend on the base of the logarithm?

If it is true for the logarithms in our list, we can use this fact to put some of them in order.

It is true that the function logxlog⁡x is increasing for all values of xx where it is defined, i.e. x>0x>0, as long as the base is greater than 11.

What if the base is between 00 and 11? For instance, the function log12xlog12⁡x? Draw the graph or work out some values if you’re unsure.

The logarithms in our list all have bases greater than 11, so

log23<log24<log25log2⁡3<log2⁡4<log2⁡5
and
log33<log34<log35 .log3⁡3<log3⁡4<log3⁡5 .

Changing bases

If instead, we take a series of logarithms of the same number but to different bases, would these increase or decrease? Think about log28log2⁡8 and log48log4⁡8. Can we use this idea to order some of our logarithms?

More generally, we are asking whether the function logxalogx⁡a is increasing or decreasing - does it depend on the value of aa? You could try using graphing software such as Desmos (find logaloga on its functions menu).

Another way to compare these logarithms of the same number would be to use the rule for changing the base:

logab=logac×logcb .loga⁡b=loga⁡c×logc⁡b .

For instance, to compare log35log3⁡5 and log45log4⁡5, we could write log35=log34×log45log3⁡5=log3⁡4×log4⁡5 and then note that log34>1log3⁡4>1, and so log35>log45log3⁡5>log4⁡5.

Estimating values

It is possible to find some other upper and lower bounds within our list of logarithms by using approximations for the square roots of 22 and 33:

We know that 212≈1.4212≈1.4, so 232=2×212≈2.8232=2×212≈2.8.

Therefore 32≈log22.832≈log2⁡2.8, and hence log23>32log2⁡3>32.

Can you use an approximate value of 3⎯⎯√3 to find an upper bound for log35log3⁡5?

We could use the following argument.
33232log35=3×312≈3×1.7=5.1≈log35.1<32332=3×312≈3×1.7=5.132≈log3⁡5.1log3⁡5<32

From this it follows that log35<log23log3⁡5<log2⁡3 which straightens out one more part of our lineup.

Increasing or decreasing again

We shall now look at two possible ways of comparing log34log3⁡4 and log45log4⁡5 which is the last unknown pairing in our lineup.

Firstly, notice that in each of these remaining logarithms, the base is one less than the number whose logarithm we are taking. We could think of them as values of the function logx(x+1)logx⁡(x+1). Is this an increasing or decreasing function? Think about its value when x=2x=2 and when xx is very large. You could use Desmos to explore it.

This appears to be a decreasing function for x>1x>1 which is the domain we are interested in. When xx gets very large, the value of this function will be very close to logx(x)=1logx⁡(x)=1 and when x=2x=2 its value is log23log2⁡3 which is certainly bigger than 11.

Different differences

Another way to compare log45log4⁡5 and log34log3⁡4 would be to work out the difference between them,

d=log34−log45 .d=log3⁡4−log4⁡5 .
If we can show that the difference is positive then we can put the two logarithms in order.

We can’t work out the difference directly, but we could try the following.
log34>log44=1so, d>1−log45but, log45>1log3⁡4>log4⁡4=1so, d>1−log4⁡5but, log4⁡5>1

This looked promising but it has only told us that dd is greater than some negative number, so it is not helpful in this case.

We can rewrite the expression without changing its value:
d=(log34−1)−(log45−1)=(log34−log33)−(log45−log44)d=(log3⁡4−1)−(log4⁡5−1)=(log3⁡4−log3⁡3)−(log4⁡5−log4⁡4)

Can you use the subtraction law of logarithms to rearrange this and show that d>0d>0?

We can complete the argument like this:
d=(log34−log33)−(log45−log44)=log343−log454d=(log3⁡4−log3⁡3)−(log4⁡5−log4⁡4)=log3⁡43−log4⁡54
 Now, log454<log354log4⁡54<log3⁡54 because logx54logx⁡54 is a decreasing function for x>1x>1. Therefore,
d>log343−log354=log3((43)(54))=log3(1615)>0d>log3⁡43−log3⁡54=log3⁡((43)(54))=log3⁡(1615)>0

So now we have shown that dd is positive, so we can put the two original logarithms in order.

Sometimes, adding zero can help you solve a problem! You may wonder how we knew that subtracting one from each side would help. The answer is that we didn’t know, but…

Looking at this numberline fragment, we already knew the two logarithms were between 11 and 22. By subtracting the ones, we have effectively compared the two red dotted lengths rather than the two blue dashed lengths. What is it that makes that feel like a worthwhile thing to do?

We have shown alternative methods for some steps of the argument. What are the advantages and disadvantages of the different methods?

Finally we have a complete lineup of logarithms.

Can you add some more logarithms to the list and use similar techniques to insert them in the lineup?
