Solution

Think about the function y=10xy=10x. Every time we increase xx by one, we multiply yy by 1010.

By how much do we multiply yy when we increase xx by 0.50.5?

To go from 101101 to 102102 we multiply by 1010. When we add to the power we multiply the expression, so to go from 101101 to 102102 in two equal steps we would add 0.50.5 to the power twice and multiply the expression by the same number twice. The number by which we multiply must therefore be the square root of 1010, such that 101+0.5+0.5=101×10⎯⎯⎯⎯√×10⎯⎯⎯⎯√=102101+0.5+0.5=101×10×10=102.

Similarly, to do it in four steps we would have to multiply by the fourth root of 1010 each time.

What does the whole number part of the power of ten tell us about the value of yy?

How is this connected with standard form?

The first column of the table is easy to fill in with whole number powers of 1010, remembering that a0=1a0=1 for any value of aa (with the possible exception of a=0a=0).

What is 0000?

Looking at the second row, when we move from 101101 to 101.25101.25 we are increasing the power of 1010 by 0.250.25, so we multiply by 10⎯⎯⎯⎯√4≈1.78104≈1.78. Moving from the first to the third column we are increasing the power by 0.50.5, which means we multiply by 10⎯⎯⎯⎯√≈3.1610≈3.16. Moving from the third to the fourth column we are increasing the power of 1010 by another 0.250.25, and so we multiply by 1.781.78.

It is clear from the table that the whole number part of the power tells us how many digits will be in the whole number part of the value of the expression. For instance, 102.anything102.anything will have three digits before the decimal point. This is a feature of powers of ten, which works because our counting system uses base-1010 numbers.

Hexadecimal is a counting system that uses base-1616 numbers and is especially useful in computer science. Here, the whole number part of xx would tell us the number of digits in the hexadecimal representation of 16x16x.

We could write 56205620 in standard form as 5.62×1035.62×103. This 33 tells us the number has four digits, which is also what the 33s in the last row of our table tell us.

Can you now find approximate solutions to the following equations?

  1. 10x=2110x=21
  2. 10x=500010x=5000
  3. 10x=56200010x=562000
  4. 10x=0.0017810x=0.00178

Approximate solutions to the first two can be found by looking in the table above. Since 2121 is between 17.817.8 and 31.631.6, xx must be between 1.251.25 and 1.51.5. We could narrow this range by trial and improvement or by plotting a graph.

562000562000 has six digits, so xx must be 5.5.something. From the right hand column of the table, the decimal part must be .75.75. This separation of the whole number and decimal parts of the number is what we do when we write the number in standard form: 562000=5.62×105562000=5.62×105.

To solve the final equation, it helps to first write the number in standard form: 0.00178=1.78×10−30.00178=1.78×10−3 so we want −3+0.25−3+0.25.

This gives us a set of approximate solutions as follows:

  1. x≈1.3x≈1.3
  2. x≈3.7x≈3.7
  3. x≈5.75x≈5.75
  4. x≈−2.75x≈−2.75

In each case, the value of xx is the power to which we have to raise 1010 in order to get the target number. This is known as the logarithm of the target number. Specifically it is the “logarithm to base 1010”, since we are working with powers of 1010. You can check your answers using the loglog or log10log10 key on your calculator.
