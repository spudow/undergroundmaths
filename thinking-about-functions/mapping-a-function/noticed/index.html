Things you might have noticed

Other multiples

What would the mapping diagrams of other functions of the form f(x)=ax+bf(x)=ax+b look like?

Using the same conventions, draw mapping diagrams for these functions. You can choose what your centre input value will be and what scale you will use.

What do you notice?

Here are mapping diagrams of our original f(x)=3xf(x)=3x together with the other functions given. We have decided to centre each of the input numberlines on zero and use a scale of one unit per tick mark; as we discovered in the Warm-up, this makes no difference to the mapping diagram, only to the labelling.

Clearly, the multiplier aa in f(x)=ax+bf(x)=ax+b makes a significant difference to the mapping diagrams:

  • f(x)=2xf(x)=2x is less “spread out” than f(x)=3xf(x)=3x, and f(x)=12xf(x)=12x results in the arrows becoming closer together on the output numberline.
  • f(x)=xf(x)=x consists of parallel arrows.
  • f(x)=−xf(x)=−x and f(x)=−2xf(x)=−2x both lead to arrows which cross themselves. (Incidentally, this is one way to see that multiplying an inequality by a negative number reverses the inequality.)

As we saw in the Warm-up, though, the mapping diagram for f(x)=ax+bf(x)=ax+b remains the same if we change the “+b+b” part (as long as we stick to our conventions).

What would f(x)=0xf(x)=0x look like?

Extending the arrows

We could extend the arrows into straight lines.

  • What patterns do you notice when you extend the arrows for different linear functions?

  • What does this tell you about the nature of linear functions?

You can explore this further using the first of the interactivities.

Here are the mapping diagrams for the same six functions as above, with extended lines.

The extended lines all meet at a single point (except for the function f(x)=xf(x)=x), which we could think of as a centre of enlargement or a focal point. This means that we can describe a linear function as a scaling of the left-hand numberline into the right-hand one.

What is the scale factor for each of the functions considered above (such as f(x)=3xf(x)=3x, f(x)=2x+1f(x)=2x+1, f(x)=12x−1f(x)=12x−1 and so on)? Can you suggest a general rule?

The scale factor for f(x)=3xf(x)=3x is 33, for f(x)=2x+1f(x)=2x+1 is 22, for f(x)=12xf(x)=12x is 1212, for f(x)=xf(x)=x is 11, for f(x)=−xf(x)=−x is −1−1 and for f(x)=−2x+2f(x)=−2x+2 is −2−2.

The general rule seems to be that the scale factor for f(x)=ax+bf(x)=ax+b is aa.

Some further questions:

  • Can you prove that every linear function behaves in this way: that it is a scaling of the left-hand numberline into the right-hand one?

  • Can you find a rule for the location of the focal point for f(x)=ax+bf(x)=ax+b?

If we can find a focal point for all of the lines, then we will have answered most of the first question as well.

The easiest case for calculations is when we have a function f(x)=axf(x)=ax (so b=0b=0), and we centre our numberlines on the arrow from 00 to 00. (And we know from earlier that this doesn’t change what the mapping diagram looks like.) If we put a coordinate system on our diagram, and imagine that our numberlines cross the xx-axis at the origin and (1,0)(1,0), then our diagram looks like this:

An arrow then goes from the point with coordinates (0,k)(0,k), which represents some value of function input, to (1,ak)(1,ak). It doesn’t even matter what the scale is on the numberlines: the arrow will still go from (0,k)(0,k) to (1,ak)(1,ak) for some value of kk.

The gradient of this line is then ak−k1−0=ak−kak−k1−0=ak−k, and so it has equation (in our “normal” xx-yy coordinate system) y=(ak−k)x+ky=(ak−k)x+k. This line will meet the xx-axis when y=0y=0, so x=−kak−k=11−ax=−kak−k=11−a. This does not depend on kk, so every line must pass through the point (11−a,0)(11−a,0).

This means that the lines all pass through a single focal point.

But we have already seen that this is not true for the function f(x)=xf(x)=x. Where does the argument fail in this case?

Using this formula, we can also say something about the location of the focal point, depending on the value of aa:

Value of aa Location of focal point
a>1a>1 to left of left-hand (input) numberline
0<a<10<a<1 to right of right-hand (output) numberline
a<0a<0 between the numberlines
a=0a=0 on the right-hand (output) numberline

This also shows that the transformation from the left-hand (input) numberline to the right-hand (output) numberline is a scaling: the scale factor is clearly just aa from the same diagram, as the line segment from (0,0)(0,0) to (0,k)(0,k) becomes the line segment from (1,0)(1,0) to (1,ak)(1,ak).

The same argument also works if we consider the more general case where we centre the diagram on a different arrow, or if we have a function f(x)=ax+bf(x)=ax+b instead. Can you see why?

Taking it further: changing the conventions

We have used several conventions in drawing our mapping diagrams. Many of them may seem sensible, but it is interesting to ask what would happen if we changed them. Possibly one of the most surprising conventions that we have used is to always make our centre arrow horizontal, by lining up the right-hand numberline appropriately.

How would things change if we did not require this? For example, what would f(x)=3x+1f(x)=3x+1 look like if we centred both numberlines at 00? You can also explore this using the second of the interactivities.

You may also like to ponder what would happen if we changed our other conventions.

Here are the mapping diagrams for f(x)=3x+1f(x)=3x+1 with the left one centred as we have been doing, and the right one with both numberlines centred at 00. For each one, we have drawn five arrows starting at −1−1, −12−12, 00, 1212 and 11.

Some observations we might make:

  • The second diagram looks fairly similar to the first in many ways.

  • The focal point is the same distance away from the left-hand numberline in both diagrams.

  • The second diagram looks just like our earlier diagrams if we imagine the left-hand numberline being centred on −12−12.

  • The second diagram looks like the first, just with the right-hand numberline moved up by 11 unit and everything else pulled along with it. (The technical name for this transformation is a shear.)

Can you explain these observations?
